# Doc Integrity Policy

Last reviewed: 2026-03-17

## Purpose

This policy defines the minimum doc structure and integrity rules for the TypeScript-only repo.

## Required docs (v1)

- `AGENTS.md`
- `ARCHITECTURE.md`
- `docs/PLANS.md`
- `docs/CHECKS.md`
- `docs/TESTING.md`
- `docs/CODE_STANDARDS.md`
- `docs/SECURITY.md`
- `docs/RELIABILITY.md`

## Required indexes

- `docs/design-docs/index.md` links to every `docs/design-docs/*.md` file except itself.
- `docs/product-specs/index.md` links to every `docs/product-specs/*.md` file except itself.

## Link integrity

- Markdown links to repo files must resolve.
- `AGENTS.md` must not contain dangling doc links.

## Generated docs

Files under `docs/generated/` must include both:

- `Generated by:`
- `Last generated: YYYY-MM-DD`

## Freshness scope

Docs that describe high-drift product behavior must include `Last reviewed: YYYY-MM-DD`.

# Doc Integrity Policy

Last reviewed: 2026-02-23

## Purpose
This policy defines the minimum doc structure and integrity rules enforced by doc-lint.

## Required docs (v1)
- `AGENTS.md`
- `ARCHITECTURE.md`
- `docs/PLANS.md`
- `docs/CHECKS.md`
- `docs/TESTING.md`
- `docs/CODE_STANDARDS.md`
- `docs/SECURITY.md`
- `docs/RELIABILITY.md`

## Required indexes (v1)
- `docs/design-docs/index.md` links to every `docs/design-docs/*.md` file except itself.
- `docs/product-specs/index.md` links to every `docs/product-specs/*.md` file except itself.

## Link integrity (v1)
- Markdown links to repo files must resolve (relative paths only).
- `AGENTS.md` must not contain dangling doc links.

## Generated docs (v1)
Files under `docs/generated/` must include both:
- `Generated by:` (tooling or source of truth)
- `Last generated: YYYY-MM-DD`

## Freshness scope (v1)
Docs that describe high-drift behavior must include a freshness marker:
- `Last reviewed: YYYY-MM-DD`
